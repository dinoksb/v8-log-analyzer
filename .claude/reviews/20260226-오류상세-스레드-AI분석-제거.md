# Code Review: ErrorDetail.tsx / page.tsx
> 리뷰 일시: 2026-02-26

## 종합 평가

**등급**: C

**요약**:
`page.tsx`에서 `ErrorEvent` 타입에 존재하지 않는 `severity` 필드를 직접 참조하고 있어 TypeScript strict 모드에서 빌드 에러가 발생합니다. 이는 즉시 수정이 필요한 Critical 이슈입니다. 반면 `ErrorDetail.tsx`는 불필요한 import 제거 및 로직 단순화가 깔끔하게 처리되었으며 컨벤션 준수 상태가 양호합니다.

---

## 발견된 이슈

### 🔴 Critical (반드시 수정)

- **[page.tsx:36]** `error.severity` 필드가 `ErrorEvent` 인터페이스에 존재하지 않음. TypeScript strict 모드에서 `Property 'severity' does not exist on type 'ErrorEvent'` 컴파일 에러.
  - 수정 방법: `ErrorEvent` 타입에 `severity` 필드 추가 또는 해당 참조 제거

### 🟡 Warning (수정 권장)

- **[page.tsx:12-22]** `getError` 함수가 Server Component에서 자기 자신의 API 엔드포인트를 fetch. storage adapter를 직접 호출하는 것이 효율적. `NEXT_PUBLIC_BASE_URL` 미설정 시 localhost fallback.

- **[page.tsx:17]** `(await res.json()) as { error: ErrorEvent }` 타입 단언이 런타임 응답 구조를 보장하지 않음.

### 🔵 Suggestion (선택적)

- **[ErrorDetail.tsx:29]** `const d = error.errorDetail` → `const detail`로 변경하면 가독성 향상
- **[ErrorDetail.tsx:105-109]** `<pre>` 안에 `<code>` 중첩 — HTML 시맨틱상 불필요

## 리팩토링 우선순위

1. **[Critical] page.tsx:36 severity 타입 오류 수정** — 빌드 차단 이슈
2. **[Warning] getError 함수 분리** — Self-referencing fetch 제거 (별도 태스크 권장)
3. **[Suggestion] const d → const detail** — 가독성 개선
