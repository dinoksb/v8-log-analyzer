## 2. 구현 계획

### 변경/삭제 파일

| 파일 | 변경 유형 | 내용 요약 |
|------|---------|---------|
| `src/components/errors/ErrorDetail.tsx` | 수정 | ThreadViewer import 제거, hasDetailReply 로직 제거, 조건 단순화, 스레드 분기 제거 |
| `src/app/errors/[errorId]/page.tsx` | 수정 | AnalysisTrigger/Card import 제거, grid 레이아웃 제거, AI 분석 Card 블록 제거 |
| `src/components/errors/ThreadViewer.tsx` | 삭제 | 스레드 UI 컴포넌트 |

| `src/components/analysis/AnalysisPanel.tsx` | 삭제 | AI 분석 결과 표시 UI |
| `src/components/analysis/AnalysisTrigger.tsx` | 삭제 | AI 분석 실행 UI |
| `src/app/api/analysis/route.ts` | 삭제 | AI 분석 실행 API |

### 단계별 실행 순서

1. **`src/components/errors/ErrorDetail.tsx` 수정**
   - `import { ThreadViewer } from './ThreadViewer'` 제거
   - `const hasDetailReply = error.thread?.replies.some(...)` 변수 제거
   - `{(d || hasDetailReply) && ...}` → `{d && ...}` 으로 조건 변경
   - `else` 브랜치("아직 로드되지 않았습니다" + RefreshDetailButton) 제거
   - 119~129줄 스레드 분기 제거 → 항상 "원문 메시지" Card 렌더링

2. **`src/app/errors/[errorId]/page.tsx` 수정**
   - `import { AnalysisTrigger } from '@/components/analysis/AnalysisTrigger'` 제거
   - `import { Card } from '@/components/ui/Card'` 제거 (미사용)
   - `<div className="grid grid-cols-1 gap-6 lg:grid-cols-2">` 래퍼 제거
   - `<Card title="AI 분석"><AnalysisTrigger .../></Card>` 블록 제거
   - `<ErrorDetail error={error} />` 단독 렌더링

3. **`src/components/errors/ThreadViewer.tsx` 삭제**

4. **`src/components/analysis/SolutionCard.tsx` 삭제**

5. **`src/components/analysis/AnalysisPanel.tsx` 삭제**

6. **`src/components/analysis/AnalysisTrigger.tsx` 삭제**

7. **`src/app/api/analysis/route.ts` 삭제**

### 아키텍처 결정 사항

- **`hasDetailReply` + else 브랜치 완전 제거**: 스레드 replies 기반 로딩 상태 표현 로직이므로 스레드 제거와 함께 제거. 결과: `{d && <Card title="오류 상세 정보">` (d 기반 렌더링만)
- **스레드 분기를 rawText 단독 렌더링으로 교체**: thread 유무와 관계없이 항상 "원문 메시지" Card 표시
- **`src/lib/types.ts` 유지**: `ErrorAnalysis`, `SlackThread` 타입이 데이터 레이어 전체에서 계속 사용됨
- **`src/lib/claude.ts`의 `analyzeError` 유지**: `analyzeErrorBatch` 내부에서 참조
- **`src/app/api/analysis/overview/route.ts` 유지**: 대시보드 기능 독립 유지

### 주의사항

- `{(d || hasDetailReply) && ...}` 의 `else` 브랜치는 RefreshDetailButton 포함 — 스레드 제거와 함께 제거
- `page.tsx`의 `Card` import 제거 필수 — 미사용 import ESLint 오류 방지
- 수정 파일 먼저 처리 후 삭제 진행 — PostToolUse lint 훅 순서 준수
- `api/analysis/overview/route.ts`는 삭제 대상 아님
