### 변경/생성 파일
| 파일 | 변경 유형 | 내용 요약 |

| `src/app/dashboard/page.tsx` | 수정 | DateRangeFetcher → SyncButton import/JSX 교체 |

### 단계별 실행 순서
1. **`src/lib/storage/adapter.ts`**: getLastMessageTs 인터페이스 추가
2. **`src/lib/storage/local.ts`** + **`src/lib/storage/supabase.ts`**: 각각 구현 (병렬 가능)
3. **`src/app/api/slack/sync/route.ts`**: 동기화 API Route 신규 생성
4. **`src/components/dashboard/SyncButton.tsx`** + **`src/app/dashboard/page.tsx`**: 컴포넌트 + 페이지 교체 (병렬 가능)

### 아키텍처 결정 사항
- **DateRangeFetcher 완전 교체**: 두 기능의 관심사가 다름. SyncButton은 자동 ts 감지, DateRangeFetcher는 수동 날짜 범위. 파일은 유지하되 dashboard/page.tsx에서만 교체.
- **getLastMessageTs를 sync API 내부에서 처리**: 클라이언트 2번 요청 대신 서버에서 원자적 처리. race condition 방지.
- **LocalFile merge 전략**: sync route에서 기존 errors 로드 → Map merge → 저장. storage 구현체 분기 없이 통일된 처리.
- **fallback 전략**: lastTs 없으면 오늘 KST 자정 Unix ts를 oldest로 사용. isFirstSync: true 플래그 반환.
- **oldest = lastTs 그대로 사용**: lastTs 이후 +epsilon 계산 없이 lastTs를 oldest로 전달. 중복은 upsert/merge가 처리.

### 주의사항
- LocalFile saveErrorEvents는 전체 교체 — sync route에서 반드시 기존 errors 로드 후 merge 필요
- Vercel 60s 타임아웃: maxDuration = 60 설정 필수
- 장기 미동기화 시 대량 메시지 타임아웃 위험 — 응답에 syncedFrom 포함하여 클라이언트에서 경고 표시 가능
- KST fallback: `new Date(\`${kstDate}T00:00:00+09:00\`).getTime() / 1000` 패턴 (기존 fetch route와 동일)

### local.ts 구현
```typescript
async getLastMessageTs(channel: string): Promise<string | null> {
  const errors = await this.loadErrorEvents(channel)
  if (errors.length === 0) return null
  return errors.reduce((max, e) => (e.ts > max ? e.ts : max), errors[0].ts)
}
```

### supabase.ts 구현
```typescript
async getLastMessageTs(channel: string): Promise<string | null> {
  const { data, error } = await this.client
    .from('error_events')
    .select('ts')
    .eq('channel', channel)
    .order('ts', { ascending: false })
    .limit(1)
    .maybeSingle()

  if (error || !data) return null
  return (data as Record<string, unknown>).ts as string
}
```
